1. Answer to the question no -01 below -->>
  null -- null মানে ইচ্ছাকৃতভাবে খালি বা কোনো মান নেই বোঝানো।
  undefined -- যখন কোনো ভেরিয়েবল declare করা হয় কিন্তু value দেওয়া হয় না, তখন তার মান undefined হয়।

2. Answer to the question no -02 below -->>
  map() -- map() একটি array method যা array এর প্রতিটি element এর উপর কাজ করে এবং একটি নতুন array return করে। সাধারণত যখন আমরা ডেটা পরিবর্তন করে নতুন array তৈরি করতে চাই তখন map() ব্যবহার করি।
  forEach() -- forEach() ও array এর প্রতিটি element এর উপর কাজ করে, কিন্তু এটি কোনো নতুন array return করে না। এটি শুধু loop চালানোর জন্য ব্যবহৃত হয়।

3. Answer to the question no -03 below -->>
   == -- == শুধুমাত্র value compare করে, type check করে না। তাই অনেক সময় এটি type convert করে ফলাফল দেয়।
   === -- === value এবং type দুটোই check করে। তাই এটি বেশি নিরাপদ এবং নির্ভুল comparison দেয়।

4. Answer to the question no -04 below -->>
   async/await -- async/await ব্যবহার করলে asynchronous কাজগুলো সহজ ও readable হয়। API call করার সময় ডেটা আসতে কিছু সময় লাগে। await ব্যবহার করলে কোড ডেটা আসা পর্যন্ত অপেক্ষা করে, তারপর পরবর্তী লাইন execute হয়।

5. Answer to the question no -05 below -->>
   scope -- Scope হলো কোনো ভেরিয়েবল কোথায় ব্যবহার করা যাবে তার সীমা।
   global scope -- Global scope এ declare করা ভেরিয়েবল পুরো প্রোগ্রামে ব্যবহার করা যায়।
   Function scope -- Function scope এ declare করা ভেরিয়েবল শুধু ওই ফাংশনের ভিতরে ব্যবহার করা যায়।
